# Phase 4: 移动端 App

> 目标：手机拍照 + 说话 + 定位 → AI 自动整理为可发布博客  
> 状态：⏳ 等待 Phase 3 稳定  
> 前置条件：Web 平台 API 稳定，有一定用户基础

---

## 核心交互流程

```
用户打开 App
    ↓
拍几张照片（餐厅/菜品/环境）
    ↓
按下录音，口述体验（"这家店的鲅鱼水饺真的很好吃..."）
    ↓
App 自动：
  - 定位：获取当前位置（附近餐厅/地点识别）
  - 语音转文字（本地 ASR，参考 Sherpa-ONNX）
  - 图片分析（识别菜品名称）
    ↓
AI 整理为草稿：
  - 标题自动生成
  - 正文结构化（餐厅信息 + 菜品 + 价格 + 体验）
  - GEO 可引用句自动插入
    ↓
用户查看/编辑草稿
    ↓
一键发布（同步到所有已配置平台）
```

---

## 技术方案

### 本地 AI（优先）

参考 blog 中 [Sherpa-ONNX 文章](../../..) 的技术路径：
- **ASR（语音转文字）**：Sherpa-ONNX 本地推理（100MB 模型，离线可用）
- **图片分析**：调用云端 Vision API（初期），或本地小模型（后期）
- **文本整理**：调用 Claude API / 本地 LLM

### 跨平台框架

- **优先**：React Native（复用 Web 代码逻辑）
- **备选**：Flutter（原生性能好，但与 Web 代码复用差）

### Computer Use 集成（用于复杂平台）

对于没有开放 API 的平台（抖音、快手），可通过：
- 桌面端 Computer Use（需连接用户的 PC）
- 手机端 ADB 模拟（Android 限定）

> 现有工具（小红书官方 App、微信）都需要用户手动编辑排版。
